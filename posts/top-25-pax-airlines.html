How's it built

The layout

The infographic uses CSS Flexbox where the smaller isotype charts are laid out in a simple Flexbox row, with flex wrap enabled. When viewed on a large display it's a 5x5 matrix of isotype charts, but on smaller displays the charts will simply wrap to the next row, while maintaining their size.

The little aircraft!

Each tiny aircraft image is the Up-Pointing Airplane Unicode glyph (&#x1F6E7;), and there are 10,016 tiny aircraft in the chart! My first version of this chart used an inline SVG for the aircraft, it worked, but it wasn't very performant. It then hit me that I could simply use an aircraft Unicode glyph, and this solution seemed to work on my desktop, but not on Android.

It turned out my default font did not support the aircraft glyph, so I had to find one that did. I eventually settled on Noto Sans Symbols 2. My only issue with this font is that it it's >300KB in size (it contained a lot of symbols). I then learnt about Font Forge, and I used this to remove all glyphs except for the aircraft, essentially creating a new single-glyph font file (4KB, yeah!).

Populating the isotypes

As mentioned above, 10,016 aircraft glyphs are used in the chart, and although I could have entered them into the HTML manually, I actually updated the dataset quite a few times while making the chart, and updating the number of aircraft for each airline (and manufacturer) would have been laborious. In order to make things a bit easier, I decided to let JavaScript (and my friend, Mendhak, help me out.)

The JavaScript function is a simple loop that adds a string of aircraft glyphs into a <span> element based on a number I enter i.e the number of aircraft from each manufacturer for each airline.

I could have used the function above to create the isotype charts, then viewed the HTML output, copied the resulting aircraft spans from the HTML output and into the source, and then removed the JavaScript (essentially pre-computing them all). And if this was the only use of JavaScript in the page then that's what I would have done, but I had to resort JavaScript (and my friend Mendhak) to build the 'view by manufacturer' option.

Toggling the 'view by manufacturer' option

I placed a "grey-fill" style in the HTML head, so this is what's used by default to fill the aircraft images, when the "assault your retinas" option is ticked, this style is removed, and the manufacturer colours are then visible.

Each airline logo is also an SVG, there are actually no bitmap images used in the chart at all. If you use your browser's zoom feature, you can zoom in on the chart and you'll notice no loss in image quality.

The data source

All data was sourced from Cirium's Ascend Fleets Analyzer. Airline fleets include only stored (AKA parked) or in-service passenger aircraft. Data was captured on 26 January 2023.
