Using 'text-wrap: balance'

In this article I show the results of enabling 'text-wrap: balance' for titles and headings across onlyrss.org.

2 Apr 2023 · 3 mins · Mark Evans

Background

If you're reading this then you are probably already aware of the upcoming new CSS feature: 

text-wrap: balance;

If not, then please check out the Chrome dev blog for the full details.

Basically, it's a way to "balance" titles, headings, and short paragraphs such that there are a similar number of words per line, and to avoid widowed words. For simplicity, I'm going to refer to it as CSS Balance from now on.

I'm not going to repeat the details that are available on the dev blog, the aim of this article is to demonstrate the effect of using CSS Balance on a typical blog, specifically this blog. Basically, I'm going to show some examples so you can decide if it's something you should implement on your website.

Examples

Homepage cards

The following examples have not been configured specifically to demonstrate the effects of CSS Balance, they are simply the result of enabling CSS Balance on this website.

Figure 1 below shows the first 3 cards from my homepage. Figure 2 shows the same 3 cards but with CSS Balance enabled.

As you can see, CSS Balance has had an effect on all 3 cards. The effect is probably most beneficial on the first first card where the words "Product" and "Manager" belong together, and on the second card, where the words "progress" and "bars" benefit from being together.

In total, enabling CSS Balance altered the title layout of 13 out of 22 cards on my homepage. This is not entirely surprising, as CSS Balance will only alter a title if that title requires more than a single line, and this is more likely where width is limited, such as in a typical card design. I wouldn't expect it to have an effect on that many titles on article pages (at least not on desktop).

Where CSS Balance is probably most useful is in stopping scenarios like that shown below in Figure 3, i.e. where a single number has appeared on a line by itself. Although to be fair, a non-breaking space should probably have been used in this scenario in order to keep the "Windows" and "10" together.

Article titles (Desktop)

I tend to use a slightly wider article page layout compared to other blogs i.e. 840px, and I don't tend to use very long article titles. Because of this, my article titles tend to only span a single line, and hence CSS Balance has no effect on desktops.

Article titles (Mobile)

On mobile (narrow displays) the situation is very different of course (it's basically the same effect as we saw on the homepage cards). Here, CSS Balance affects many titles. As we can see before in Figure 4.

In the first example below, where CSS Balance is NOT used, we see that as the width is reduced, the last word simply drops to a new line, as you would expect. This is a good example of why CSS Balance is useful, without it "meters" becomes disconnected from "million", which then becomes disconnected from "15", and these 3 really do belong together

With CSS Balance enabled, there is far less movement of the title as the screen width is reduced, and the phrase "15 million meters" remains together.
